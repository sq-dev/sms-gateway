Short reference for the `osonsms` adapter in `sqdev/sms-gateway`.

## Summary

- class: `SmsGateway\Providers\OsonSms\OsonSmsProvider`
- provider name: `osonsms`
- capability: `send` and `status`
- default base URI: `https://api.osonsms.com`
- auth: `Bearer` token + `login`

The adapter implements `SmsProviderInterface`, so it supports both sending and `getStatus()`.

## Plain PHP

```php
use SmsGateway\Providers\OsonSms\OsonSmsProvider;
use SmsGateway\Sender;

$sender = new Sender(new OsonSmsProvider(
    token: $_ENV['OSONSMS_TOKEN'],
    login: $_ENV['OSONSMS_LOGIN'],
    defaultSenderName: 'MYBRAND',
));

$result = $sender->send('+992900000000', 'Hello');
```

## Laravel config

```php
'osonsms' => [
    'driver' => 'osonsms',
    'token' => env('OSONSMS_TOKEN'),
    'login' => env('OSONSMS_LOGIN'),
    'default_sender_name' => env('OSONSMS_DEFAULT_SENDER'),
    'base_uri' => env('OSONSMS_BASE_URI'),
],
```

## Important: composite `messageId`

For status lookups, OsonSMS needs two values:

- `txn_id` generated by the client
- `msg_id` assigned by the server

The adapter encodes both into a single `messageId`:

```text
{txn_id}|{msg_id}
```

Treat this value as opaque. Do not parse it manually and do not construct it yourself. Pass the same `messageId` you got from `send()` into `getStatus()`.

## Supported metadata

| Key | Meaning |
| --- | --- |
| `txn_id` | your own idempotency key |
| `channel` | optional extra channel such as `telegram` |
| `is_confidential` | when `true`, the provider does not store the message text |

If you provide `txn_id` manually, it must be a non-empty string and must not contain `|`.

## Status tracking

```php
use SmsGateway\Contracts\TracksSmsStatusInterface;

$provider = $sender->provider();

if ($provider instanceof TracksSmsStatusInterface) {
    $status = $provider->getStatus($result->messageId);
}
```

Core status mapping:

| OsonSMS | MessageStatus |
| --- | --- |
| `ENROUTE` | `Queued` |
| `ACCEPTED` | `Sent` |
| `DELIVERED` | `Delivered` |
| `EXPIRED` | `Expired` |
| `UNDELIVERABLE` | `Undelivered` |
| `DELETED`, `REJECTED` | `Rejected` |
| anything else | `Unknown` |

## Important behavior

- phone numbers are cleaned by stripping `+`, spaces, dashes, and parentheses; the adapter does not auto-add a country code
- both send and status requests use `GET`
- retrying with the same `txn_id` can be idempotent after timeouts
- HTTP, auth, and transport failures are translated into `ProviderException`
